# lint 技能：知識圖譜健康巡檢

## 核心目標
將軟體工程中的「靜態程式碼分析」引入知識管理。定期執行此 skill，找出知識庫長期演進中產生的：死連結、孤立頁面、未同步索引、認知衝突。

## 觸發條件
- 使用者輸入 `/lint`
- 使用者詢問「我的知識庫健康狀況如何」
- 使用者要求「檢查知識庫狀態」或「檢查健康」

## 知識庫路徑
- 使用 Glob 工具動態定位當前工作區下的 wiki/ 目錄

## 巡檢流水線

### 第 1 步：索引一致性檢查
1. 讀取 `wiki/index.md` 全部內容
2. 掃描 `wiki/` 下所有 `.md` 檔案（排除 index.md 和 log.md）
3. 提取 index.md 中登錄的所有雙連結 `[[頁面名稱]]`
4. 比對：找出已登錄但檔案不存在的條目，或檔案存在但未登錄的頁面

### 第 2 步：雙向連結健康檢查
1. 掃描所有 `.md` 檔案，提取所有 `[[雙連結]]` 格式的連結
2. 如果連結指向的頁面不存在 → 標記為**死連結**
3. 統計被引用的頁面（排除 self-reference）
4. 找出從未被任何其他頁面引用的頁面 → **孤立頁面**

### 第 3 步：認知衝突審查
1. 全域搜尋所有 `.md` 檔案中包含 `## 知識衝突` 的頁面
2. 提取每個衝突的簡要描述（衝突雙方是什麼）
3. 統計帶未解決衝突的頁面（認知技術債）

### 第 4 步：收件匣積壓檢查（可略過）
- 本 skill 僅掃描 wiki/，略過 raw/ 收件匣檢查

## 報告輸出規範

掃描完成後，輸出結構化報告，嚴格遵循以下格式：

```markdown
## 🩺 知識庫健康體檢報告 — YYYY-MM-DD

### ✅ 綠燈項
- [運行良好的項目]

### ⚠️ 黃燈項
- **發現 N 個孤立頁面**：[清單] — 建議新增關聯或分類
- **發現 N 個未同步索引**：[清單] — 檔案存在但未在 index.md 登錄

### ❌ 紅燈項
- **發現 N 個死連結**：[來源頁面] → [[不存在的目標頁面]]
- **存在 N 個未解決的知識衝突**：[頁面名稱]

### 🛠️ 下一步行動
1. 是否需要自動修復未同步索引？
2. 是否需要針對知識衝突進行重新推演？
```

## 強制限制
- **僅讀掃描**：生成報告前，禁止修改、刪除、重新命名任何檔案
- **手動確認**：報告後等待使用者確認再執行修復
- **靜默日誌**：修復完成後，在 wiki/log.md 追加 `## [YYYY-MM-DD] lint | 修復了 N 個問題`
