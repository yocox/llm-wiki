# query 技能

## 核心目標
將使用者的提問轉化為對本地 Wiki 的深度檢索。提取相關頁面資訊，綜合出帶有明確引用來源的雙連結回答。當回答具有高價值時，主動將其固化為知識庫的一部分。

## 觸發場景
- 使用者輸入 `/query <問題>`
- 使用者用自然語言詢問：「我的筆記裡關於 X 是怎麼說的」、「過去我對 Y 的決策是什麼」、「查詢 Z 相關的知識」
- 使用者提及 wiki、知識庫、筆記、記錄等關鍵字

## 降級策略
如果問題屬於純通用知識（如「太陽系有幾顆行星」），且 wiki/index.md 中無相關內容：
> 本地知識庫中未找到相關內容，以下為通用知識回答：[直接回答]

---

## 檢索與綜合流水線

### 步驟 1：查閱全域索引
**永遠的第一步**：讀取 `wiki/index.md`

在 index.md 中定位與問題相關的：
- Entities（實體）
- Concepts（概念）
- Sources（摘要）
- Syntheses（綜合）

### 步驟 2：深度閱讀目標檔案
選取步驟 1 中找到的最相關頁面，使用讀取工具取得完整內容。

### 步驟 3：綜合與回答
綜合資訊，回答使用者問題。

**雙連結引用規範**：
- 每當引用某個 Wiki 頁面的資訊，在文字中使用 `[[頁面名稱]]` 標注
- 整段引用同一頁面：段落首尾各引用一次
- 引用特定原文：使用 Markdown 區塊引用 `> 引用內容`

### 步驟 4：高價值內容固化
如果滿足以下條件，主動詢問使用者是否儲存為 synthesis：
- 回答超過 2 個段落
- 內容具有分析對比性或總結性

詢問話術：
> 這是一個有價值的總結，是否需要我將其儲存到 wiki/syntheses/ 目錄？

使用者同意後，按照 CLAUDE.md 規範建立檔案：
```markdown
---
title: "頁面標題"
type: synthesis
tags: []
sources: []
last_updated: YYYY-MM-DD
---

# 總結內容
```

並在 `wiki/index.md` 的 Syntheses 分類下登錄。

### 步驟 5：記錄操作日誌
無論是否生成 synthesis 頁面，查詢結束後必須在 `wiki/log.md` 末尾追加：

```markdown
## [YYYY-MM-DD] query | <操作簡述>
- **輸出**: <引用頁面清單或「即時回答未儲存」>
```

格式必須完全遵循 CLAUDE.md 中的範例。

---

## 強制約束
- **禁止憑記憶回答**：必須先檢索知識庫
- **禁止過度引用**：同一頁面的資訊在段落首尾引用一次即可
- **禁止靜默回答**：知識庫無相關內容時必須聲明
